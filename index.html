GpLua2x

Introduction

GpLua2x is a port of the Lua scripting language, including a small wrapper to access some functions of SDL, for the GP2x.

Download

gplua2x-VERSION.tar.gz

Lua Interpreter

The binary gplua2xrun can be used to run stand-alone Lua applications/games on the GP2x.

Embedded Lua

If you just want to use Lua to do some scripting within an application mainly written in C or C++, you can link with the included liblua.a for GP2x. For testing on your development system an ordinary Lua 5.1 installation can be used instead. If you use some of the functions of the SDL wrapper you also need to link with libsdl and libsdl-image from the open2x project.

Compiling for Linux

To compile for a Linux development host, you need to have development libraries for Lua 5.1, SDL 1.2 and SDL-image 1.2 installed. Run make linux.

Compiling for FreeBSD

Similar to if you want to develop on Linux, for FreeBSD you need to install the ports of Lua 5.1, SDL 1.2, and SDL-image 1.2, then compile using gmake freebsd.

Compiling for GP2x

Compilation for the GP2x is easiest to do with the included Makefile and the toolchain and libraries from the open2x project, and the source for Lua 5.1.2. The Lua sources should be placed in a directory called lua in the directory with the gplua2x files. You need to patch Lua using the included diff file to make it compile properly for the GP2x. Run make gp2x. Note that binaries for the GP2x are included. There is no need to compile your own unless you modify the source (gplua2x.c and related files, or Lua itself).
